# ScoreSync

A fully responsive, Firebase-powered cross-platform sports scoring and match-tracking application built with Flutter.

## Firebase SDK Integration via FlutterFire CLI

This project has been seamlessly integrated with multiple Firebase SDKs using the official FlutterFire CLI tool to auto-generate the credentials and link Android, iOS, and Web targets simultaneously without manual Gradle/Plist configuration.

### Steps Performed & Configuration Commands:
1. **Installed Firebase CLI:**
   ```bash
   npm install -g firebase-tools
   ```
2. **Installed FlutterFire CLI globally:**
   ```bash
   dart pub global activate flutterfire_cli
   ```
3. **Authenticated with Firebase Account:**
   ```bash
   firebase login
   ```
4. **Configured FlutterFire (Target Registration):**
   ```bash
   flutterfire configure
   ```
   *Selected the Active Firebase project and checked target platforms (Android, iOS, Web).*
5. **Added SDK Dependencies in `pubspec.yaml`:**
   ```yaml
   dependencies:
     firebase_core: ^3.0.0
     firebase_auth: ^5.0.0
     cloud_firestore: ^5.0.0
     cloud_functions: ^5.0.0
   ```
6. **Initialized Firebase with Generated Credentials in `main.dart`:**
   ```dart
   import 'package:flutter/material.dart';
   import 'package:firebase_core/firebase_core.dart';
   import 'firebase_options.dart'; // Auto-generated by CLI

   void main() async {
     WidgetsFlutterBinding.ensureInitialized();
     await Firebase.initializeApp(
       options: DefaultFirebaseOptions.currentPlatform, // Maps configurations dynamically
     );
     runApp(MyApp());
   }
   ```

### Reflection
* **How does FlutterFire CLI simplify Firebase setup?**
  It dramatically cuts down setup time by avoiding manual edits to deeply nested Android Gradle builds or iOS `Info.plist` files. It creates a single source of truth in the `firebase_options.dart` file that manages keys for multiple platforms instantly based on the runtime target.
* **What issues did you face and how did you resolve them?**
  Ran into path resolution errors initially where `flutterfire` was not recognized in the terminal globally. Solved it by exporting `~/.pub-cache/bin` explicitly into my System's `$PATH` variable so the global dart package binaries could execute.
* **How will this help your team integrate more Firebase features later?**
  Any future plugin like Firebase Storage or FCM (Messaging) added to the `pubspec.yaml` will now inherently inherit the bindings configured by the single CLI file automatically. We won't ever have to manage separate configurations again.
